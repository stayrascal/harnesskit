# ADR-000: ADR Template

## Status
Accepted

## Context
We need a consistent format for recording architectural decisions in harnesskit.

## Decision
Use Markdown ADR format with sections: Status, Context, Decision, Consequences.

File naming: `NNN-title.md` (e.g., `001-template-engine.md`).

## Consequences
- All significant architecture decisions are documented
- New contributors can understand the reasoning behind choices
- Decisions can be revisited with full context
